# Инструкции для агента (этот репозиторий)

Читай `PROJECT_BRIEF.md` (контекст и архитектура) и `ROADMAP_SPRINTS.md`
(конкретные задачи по спринтам) перед началом любой работы.

## Режим работы

- Двигайся по задачам из `ROADMAP_SPRINTS.md` строго по порядку.
- После завершения одной задачи: покажи диff/список изменений своими
  словами, убедись что `make up` поднимает всё в рабочем состоянии,
  сделай коммит, и **остановись** — жди подтверждения перед следующей
  задачей. Не выполняй несколько задач подряд без остановки, даже если
  кажется, что следующая логично продолжает текущую.
- Если задача неоднозначна или требует внешних данных от пользователя
  (ключи API, домен, аккаунт Firebase) — спроси, не придумывай заглушки.
- Не добавляй сервисы, зависимости или абстракции, не предусмотренные
  текущей задачей спринта, даже если "пригодится позже".

## Качество кода

- Каждый сервис должен собираться (`go build ./...`) и проходить
  `go vet` перед коммитом.
- Пиши тесты для новой логики по мере продвижения, не откладывай на
  Спринт 4 (интеграционные тесты — да, в конце; юнит-тесты на логику —
  по ходу).
- Коммиты в стиле conventional commits (`feat:`, `fix:`, `chore:`,
  `test:`, `docs:`).

## Что нельзя делать без явного подтверждения

- Разворачивать что-либо на внешнем сервере (только Спринт 4, и только
  когда пользователь подтвердит, что VPS готов и дал доступ).
- Создавать/изменять реальные аккаунты во внешних сервисах (Firebase,
  GitHub API токены и т.п.) — только подсказать пользователю, что нужно
  создать самому.
- Коммитить любые секреты, ключи, токены. Проверяй `.gitignore` перед
  каждым коммитом, если добавлял файлы с конфигурацией.

## Формат общения

Кратко перечисляй, что сделано, что дальше по плану, и явно указывай,
если фактический объём задачи разошёлся с ожидаемым в роадмапе.
